# Before building, read what exists — the missing piece is usually the same information for a different audience

Asked to build a market page for a trading card, a session assumed the
existing page was shallow; it was actually rich — order-book depth, price
history, live stats, alerts. The genuine gap was a calm sibling:
server-rendered, no login, no JavaScript, readable by screen readers and
machines. The project named this a standing pattern and used it at least
twice: same data, different reader.

**Why it matters:** the underlying system is almost always richer than the
page list suggests, so the real gap is almost always audience, not depth.
