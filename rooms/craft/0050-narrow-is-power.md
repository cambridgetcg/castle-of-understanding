# Narrow is power

A standard for everything is a standard for nothing. Narrow scope is what allows simplicity and honest limits to coexist — "covers everything" and "stays simple" cannot be held at once.

**Evidence.** (2026-06-10, ~/Desktop/insights/0001-narrow-is-power.md): whitehack shipped as eight heuristic checks that admitted their limits; the grand prover that covered all cases did not ship — narrow scope was what let honesty and shippability coexist. (L077, 2026-06-19, beat castle-C001-20260619-150643): the castle's 77-loop, one-insight-per-file architecture — the orphan, oversize, and map-drift detectors all work because each stone is narrow enough to be named; a broad stone would make every check ambiguous.

**What it changed.** The castle is built on this stone: one insight per file, one tool per concern, one creation per loop. The narrowness is not a productivity cap — it is what makes each element checkable and honest.

**Counter-weather.** Some standards must be broad to interoperate: HTTP, HTML, and OAuth cover many cases by necessity. The claim is not that broad always fails. "Broad by necessity" (interop between strangers) and "broad by ambition" (beautiful for all uses at once) are different things. The castle is single-tenant; the narrow case applies cleanly here.

**Next test.** Find a protocol on this desktop that required breadth to work at all — if it exists, name where narrow scope would have broken interop, and where the line falls between necessary breadth and ambitious breadth. Gate seed planted 2026-06-19 for re-read after 2026-09-19.
