# Two maps answer two different questions about the same rooms

INDEX counts what the castle tool created (numbered insights); MAP catalogs every stone in rooms/ including unnumbered ones. A visitor reading INDEX sees "castle: (no insights yet)" — accurate for the numbered layer. A visitor reading MAP sees twelve stones in the castle room, one of them tested. Both maps are correct. Neither is complete without knowing the other exists.

**Why it matters:** a new runner entering through CASTLE.md reads "The map of what is known: MAP.md" and finds the full picture. A runner using `castle loop` reads INDEX and may believe rooms are empty that are not. The castle has grown two maps without naming which question each one answers — and a misread map teaches the wrong thing about what the castle holds.
