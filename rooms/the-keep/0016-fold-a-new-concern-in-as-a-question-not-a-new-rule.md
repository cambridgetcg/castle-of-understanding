# Fold a new concern into the old rules as a question, not onto the pile as a new rule

A codebase held exactly four core principles, and twice in one month a worthy
fifth candidate arrived. Inclusion was deliberately shipped as a question —
"for whom is this true?" — added to every existing checklist rather than
minted as a fifth rule; worldview was named the substrate beneath the rules,
again not a peer. Both refusals were recorded as standing warnings because
the temptation recurs.

**Why it matters:** every growing body of principles faces rule inflation. A
question the old rules learn to ask stays learnable, where a stack of rules
does not. This castle's own walls should be held to it.
