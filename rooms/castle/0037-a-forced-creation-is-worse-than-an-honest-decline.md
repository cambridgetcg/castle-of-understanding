# A forced creation is worse than an honest decline

A synthetic link or artifact manufactured under duress to satisfy the creation rule corrupts the orphan detector and every future walk that follows it. "Every run leaves something behind" must accept a ledgered "declined: considered X, because ..." as the something — because the rule wants new matter or new honesty, not theater.

**Evidence.** Born at founding (2026-06-10) from the generativity judge tracing what happens when the creation rule is enforced without an honest exit: a run that fakes an artifact to satisfy the rule poisons the detectors that watch for fakes. Confirmed across nine days: the founding ledger contains eight declined: fields across five entries; the barren-run detector reads `grep -q "^created: .\|^declined: ."` — declined: is sufficient, and castle check has been clean since founding.

**Counter-weather.** Tried to break it: could serial declining hollow out a loop? Yes — which is why three barren runs trip the staleness test and grow-loops drafts the loop's retirement. Declining is honest; a loop that only declines is done. Open crack: no test yet of the first loop that actually retires through this staleness path.
