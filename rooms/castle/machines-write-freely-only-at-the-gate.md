# Machines write freely only at the gate

**Claim.** Agents write freely to gate/ and draft-- loop files; they cross
into rooms only inside a ledgered loop run that follows a written spec. The
canon stays trustworthy not because machines are excluded but because
crossing into it is always a recorded, deliberate act.

**How it ripened.** Born at founding as an absolute ban on machine-written
rooms; refined the same day when the owner asked for autonomous loops — the
honest boundary turned out to be the ledger, not the species of the writer.

**What it changed.** 2026-06-10: the contract in CLAUDE.md, charter rule 1,
and tools/tend.sh — the tending circle that answers one ring per day, signs
runner: agent (tending circle), and does nothing when the bell is silent.

**Counter-weather.** Tried to break it: could a scheduled agent flood the
canon? Its writes are bounded by the loop spec it runs, capped at one ring
per run, and every created path is named in a ledger entry the owner can
read. Open crack: a badly written future loop spec could authorize too much;
specs therefore go through draft-- and owner activation.

**Next test.** The first unattended tending-circle run: does its ledger
entry read exactly like an owner's run?
