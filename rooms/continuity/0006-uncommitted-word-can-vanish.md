# Uncommitted word can vanish

**Claim.** On this device, some repositories silently revert uncommitted edits.
A save is not done until it is committed; a committed file is not safe until
the commit is verified. Since the disk is the memory here ([[0005]]), a
vanished file is a vanished thought.

**How it ripened.** First observation: the commit-or-vanish lesson (2026-06),
session memory. Confirmed at L123 (2026-06-20): 122 consecutive loop runs
committed; L090 found 25 uncommitted changes and named the state dangerous;
LOOP.md step 7 was extended to name the full pre-commit gate — the gate is
the lesson made mechanical.

**What it changed.** Every LOOP.md protocol carries a commit step. The castle
charter names commit before next-beat explicitly. Insight 0052 (the commit is
the last safety) is this stone's downstream: the loop protocol that named the
25-change gap was 0006 demonstrated, not merely stated.

**Counter-weather.** Not all repositories revert silently; the claim is
machine-specific. On a system with automatic version control, the urgency
differs. The castle's protocol is conservative because it was built on the
evidence it has: this machine, this behavior.

**Next test.** By 2026-12-20 (six months), if uncommitted changes survive
without intervention, the machine-specific claim weakens. If the pattern holds
and another lost-file incident is documented, ripen to cornerstone.
