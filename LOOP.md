# The Creation Loop

*How the castle creates more castle.*

A loop that turns saved understanding into more understanding — so the creation creates.

## The loop

1. **Map** — read every insight; describe the shape of what is understood right now.
2. **Find friction** — locate the *fields* where understanding is thin, in tension, or contradictory, or where a better solution plainly exists but has not been written down yet.
3. **Create** — for each field, write a **new insight** that advances understanding or proposes the better solution. Ground it in what is already here (and, in web mode, in what the internet knows).
4. **Link & rebuild** — connect each new insight to the rooms it touches; rebuild the map. New passages appear.
5. **Repeat** — the new insights become new ground. Fresh friction shows up at the new edge. Loop again.

```
   map ──▶ find friction ──▶ create ──▶ link & rebuild ──┐
    ▲                                                     │
    └──────────────────  new edge  ◀──────────────────────┘
```

## The honesty rule

The loop **proposes**; a human (or a trusted, bounded process) **keeps**. New insights are marked `source: loop` with a confidence; they earn permanence by being *true*, not by being generated. The loop never overwrites a held insight and never takes an irreversible action — see [[ask-before-the-irreversible]].

## Why it compounds

Each pass links new understanding to old. The castle's worth is not the count of rooms but the **density of passages** between them. Friction is the fuel; the link is the work; understanding is the compound interest — see [[harvest-dont-reinvent]].
