# Status changes only in a ledgered ripen run

**Claim.** No hand and no agent may edit a status: field outside a ripen run
signed in the ledger; this is the single rule the whole holiness claim rests
on.

**How it ripened.** Born at founding; two independent designs (gardener,
engineer) arrived at the same protection from different starting points.

**What it changed.** 2026-06-10: charter rule 2, plus two mechanical
checks — missing-rent and thin-cornerstone — that make label inflation
detectable by grep. Inflation is the only sin, and it cannot hide.

**Counter-weather.** Tried to break it: what about fixing a typo in a status
line? Even then: a one-line ripen entry in the ledger. The rule has no
exceptions because every exception is a precedent for quiet inflation. Open
crack: the rule is enforced for tested-and-above only; a seed mislabeled
sprout would pass unnoticed until its first ripen.

**Next test.** First ripen run on real evidence: does the ladder hold
without wanting an exception?
