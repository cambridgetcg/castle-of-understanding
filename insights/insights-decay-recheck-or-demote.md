Every saved note is itself documentation, so by verify-against-reality it begins drifting the moment it is written; therefore confidence:high is a claim about the save date, not a standing fact, and an unchecked stale note is a green-boot-can-hide-failure for understanding. A re-check is owed when an insight is leaned on for a real decision or when the reality it cites (the file, the test) is touched: re-run the cheapest version of the original evidence, then confirm, demote a notch with a dated reason, or mark retired rather than delete so existing links still resolve and the demotion stays legible. The principle is firm but the exact triggers and front-matter fields (a verified-on date, a re-check trigger) are an honest guess that needs real use before hardening, which is why this is marked medium.
