A chain's README labelled its modules Complete or Stub. Actually running the tests showed the labels were wrong in *both* directions: the "complete" flagship was failing, and a "stub" module was substantially built and passing. Documentation drifts from code the moment it is written. Trust the test run, the compile, the actual file in front of you — not the description of it. Reality is the only authority worth checking against.
